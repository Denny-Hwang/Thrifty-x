# Raspberry Pi 5 배포 분석 및 실행 계획 (독립형 RX Site)

작성일: 2026-05-06  
대상 저장소: `Thrifty-x`

## TL;DR (핵심 결론)

- **배포 가능성:** 높음 (코드 구조/의존성 측면에서 Pi 5 구동 가능)
- **즉시 운영 가능성:** 제한적 (장시간 무인 운용 검증 미완료)
- **최종 판단:** **"실기기 검증 완료 전까지는 문제없음 보장 불가"**

즉, 지금은 “동작 가능 상태”이며, “현장 무중단 운영 보장 상태”로 만들기 위해 아래 개선/검증 작업이 필요하다.

## 1) 현재 상태 요약

### 1.1 코드/문서 상 강점
- Python 3.10+ 및 `pyproject.toml` 기반 패키징으로 배포 재현성이 양호함.
- Airspy Mini/R2 옵션(샘플레이트, gain, serial 지정, packing 등)이 이미 CLI에 노출됨.
- Linux 장치 권한(udev/plugdev) 및 장치 점유 충돌 주의점이 README에 문서화됨.

### 1.2 현재 갭
- `rpi/installation.md`는 Pi 3/Jessie 중심 문서로 Pi 5 기준과 불일치.
- Pi 5용 표준 설치/운영 절차(서비스 자동기동, 로그정책, 헬스체크)가 분리되어 있지 않음.
- 장시간 안정성/복구성(재부팅, 네트워크 단절, 저장공간 부족) 검증 결과가 없음.

## 2) Pi 5 배포 위험요소와 개선 포인트

### 2.1 하드웨어/IO
- USB 캡처 안정성(특히 고샘플레이트)은 현장 환경(케이블/전원/온도)에 민감.
- SD카드 기반 장기 write workload는 수명 리스크가 큼.

**개선 포인트**
- `--packing` 기본 적용 여부를 현장 시나리오별 검토.
- 저장정책(보존기간, 압축, 순환삭제) 강제 적용.
- 권장 전원 어댑터/케이블 스펙 문서화.

### 2.2 소프트웨어/의존성
- `libairspy` 설치 방식(apt/source) 차이로 초기 설치 실패 가능.
- `pyfftw`는 선택 의존성으로 유지하는 것이 안전.

**개선 포인트**
- Pi 5 전용 설치 문서에서 “최소 설치 경로”와 “성능 확장 경로”를 분리.
- 설치 검증 명령(버전, import, 장치열거) 체크리스트화.

### 2.3 운영/복구
- 무인 노드 운영 핵심은 자동복구(systemd), 관측성(log/heartbeat), 원격접속.

**개선 포인트**
- 서비스 템플릿 제공 (`thriftyx-capture@.service` 등).
- 로그 제한/journal 정책 및 장애 대응 런북 표준화.

## 3) 작업 계획 (우선순위 + 산출물)

## P0 — 배포 가능 상태 확보 (필수)
1. **Pi 5 전용 설치 문서 작성**
   - 산출물: `rpi/installation_pi5.md`
   - 포함: OS, 필수 패키지, libairspy, pip install, udev, device serial 확인
2. **실행 검증 체크리스트 작성**
   - 산출물: `docs/rpi5_validation_checklist_ko.md`
   - 포함: 캡처 60초, detect, 재부팅 후 자동기동
3. **systemd 기본 템플릿 추가**
   - 산출물: `rpi/systemd/*.service` 예시

## P1 — 안정성 확보 (권장)
1. **24시간 soak test 프로토콜 문서화**
2. **스토리지/로그 정책 문서화**
3. **장애 대응 런북 추가**

## P2 — 운영 확장 (선택)
1. 노드 표준 설정 템플릿/환경변수 전략
2. 원격 업데이트/롤백 절차
3. heartbeat/metrics 최소 스키마 정의

## 4) 테스트 필요사항 (필수/권장)

### 4.1 필수 테스트 (Go/No-Go)
- [ ] 장치 인식: Airspy serial 열거 성공
- [ ] 60초 캡처: 프로세스 비정상 종료 없음
- [ ] detect 산출물 생성 확인
- [ ] 재부팅 후 서비스 자동 시작 확인
- [ ] 2시간 연속 운용 시 메모리 누수/크래시 없음

### 4.2 권장 테스트
- [ ] 24시간 soak test
- [ ] 저장공간 90% 근접 상황에서 정책대로 순환 삭제
- [ ] 네트워크 단절/복구 시 timestamp 품질 유지
- [ ] 다중 Airspy 연결 시 serial 지정 정확성

### 4.3 합격 기준 (초안)
- 필수 테스트 100% 통과
- 24시간 테스트 중 치명 장애 0건
- 데이터 유실률/샘플 드롭률이 현장 허용치 이하

## 5) “문제없이 돌아간다” 판단 기준

다음 3가지를 만족하면 “문제없이 운영 가능”으로 판단한다.
1. 기능: 캡처/탐지/업로드 파이프라인 정상
2. 안정성: 24시간 이상 무인 동작, 자동복구 성공
3. 운영성: 로그/경보/원격접속/복구절차 문서화 완료

현재는 **1번 일부만 확인된 상태**로 간주하는 것이 안전하다.
